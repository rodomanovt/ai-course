# HW14 – эмбеддинги, FAISS, оценка retrieval и mini-RAG по базе знаний

## 1. Кратко: что сделано


Выбранная база знаний представляет собой статьи из русскоязычной Википедии по предметной области "Искусственный интеллект". Некоторые из тем статей пересекаются. По данным статьям можно задать много вопросов, так как текст являются достаточно содержательными. Чанкинг организовывался разбиением документов на строки по chunk_size слов с overlap. Использовался SentenceTransformer и FAISS. Базовая проверка retrieval показала hit@k = 1 для всех тестовых запросов. После добавления новых документов retriever стал опираться на них при специфических запросах. Mini-RAG хорошо справляется с вопросами, ответ на которые напрямую находится в исходных текстах. Если задать более общий вопрос, ответ на который нужно собрать из нескольких документов, то RAG начинает путаться. Для исправления данной проблемы можно попробовать увеличить размер чанка и оверлап. Если задать вопрос, не относящийся к тематике документов, то RAG выдает ответ с похожими по смыслу словами с очень низкой релевантностью.

## 7. Анализ

Это связано с тем, что в retriever передается много лишнего контекста. При обновлении базы знаний стали выдаваться ответы из новых документов с большей релевантностью. Mini-RAG хорошо справляется с вопросами, ответ на которые напрямую находится в исходных текстах. Если задать более общий вопрос, ответ на который нужно собрать из нескольких документов, то RAG начинает путаться. Для исправления данной проблемы можно попробовать увеличить размер чанка и оверлап. Если задать вопрос, не относящийся к тематике документов, то RAG выдает ответ с похожими по смыслу словами с очень низкой релевантностью.

## 8. Итоговый вывод

Для данной задачи достаточно простого retriever на основе SentenceTransformer и FAISS. Retriever оказывает прямое влияние на итоговое качество RAG, так как из его ответов строится итоговый ответ. Качество retrieval можно экспериментально регулировать, изменяя размер чанка или оверлап.
