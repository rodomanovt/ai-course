# HW12 – временные ряды: temporal split, baseline-модели и GRU-прогноз

## 1. Кратко: что сделано

В работе использовался датасет `S12-hw-dataset.csv`, который представляет собой временой ряд. На основе предыдущих значений переменной предсказывалось следующее значение. `train / validation / test` организовывался с помощью temporal split, разбивая данные на 3 отрезка, идущие подряд. Сравнивался наивный бейзлайн, скользящее среднее, модель Rigde и GRU. GRU показал наилучший результат по метрикам.

## 2. Среда и воспроизводимость

## 3. Данные и постановка задачи

- Датасет: `S12-hw-dataset.csv`
- Целевая колонка: `target`
- Частота временного ряда: 1 измерение в час
- Горизонт прогноза: 1
- Размер датасета: 4320
- Диапазон дат: 2025-01-01 00:00:00 - 2025-06-29 23:00:00
- Есть ли дополнительные внешние признаки: нет
Ряд имеет восходящий тренд с периодическими недельными повышениями значения target. Присутствует шум и резкие скачки.

## 4. Temporal split и признаки

### 4.1. Разбиение по времени

- Как именно выполнен `train / validation / test`: `train / validation / test` организовывался с помощью temporal split, разбивая данные на 3 отрезка, идущие подряд.
- Какие доли или интервалы использовались: 70 / 15 / 15
- Почему `random split` здесь некорректен: Random split некорретен для временных рядов, потому что он перемешивает значения. Моделоь будет учиться на смешанных данных из прошлого и будущего, что не соответствует задаче предсказания будущего используя только прошлое.


### 4.2. Признаки для baseline-моделей

- Какие лаги использовались: 1, 7, 14
- Какие rolling-признаки использовались: rolling_mean_7, rolling_std_7
- Какие календарные признаки использовались: day_of_week
- Как обрабатывались пропуски после построения лагов/rolling: начало датасета обрезалось до того момента, пока не стали определены все лаговые признаки 
- Использовалось ли масштабирование и где именно оно обучалось: для Ridge использовался StandartScaler

## 7. Анализ

Наивный бейзлайн оказался достаточно сильным. Moving average и лаговые признаки не дали прироста качества относитьельно naive last. Ridge сработал хуже, чем naive last, но значительно лучше, чем moving average. Это может быть связано с тем, что в данных много шума. Поэтому moving average показал себя хуже всего. GRU помогла на данном ряде, качество ощутимо улучшилось по сравнению с бейзлайнами. Ряд имеет четкий восходящий тренд и сезонность, это помогает в прогнозе. Утечки данных могут появиться, если для вычисления дополнительных признаков использовать данне из будущего, либо сделав случайное разбиение на train / val / test. Мы выбираем лучшую модель по validation, так как test используется для финальной оценки лучшей модели. Test содержит данные, которые модель никогда не видела и не использовала для обучения. Модель не смогла предсказать выбросы, как видно по [графику](./artifacts/figures/best_forecast_test.png)


## 8. Итоговый вывод

Для данной задачи лучше подходит GRU, так как она показала наилучшую ошибку. В рядах с меньшим количеством шума наивный baseline будет работать достаточно хорошо. GRU и нейросети имееют смысл тогда, котгда в ряде нет глобальных трендов и сезонности. Самое главное во временных рядах - это не допустить того, чтобы модель обучалась предсказыванию на данных из будущего.
